为了让你完全明白，我们需要区分清楚大语言模型的两种使用方式：**模型训练（微调 Finetuning）** 和 **提示词工程（Prompting / In-context Learning）**。

在论文的这一部分（3.2 节的 ReAct Prompting），作者**完全没有修改模型的任何参数**，模型（PaLM-540B）被称作是“冻结的（frozen）”。

那么，“人工改写成了标准的 ReAct 格式”到底是在干什么呢？答案是：**编写供模型参考的“模板文本（Prompt）”。**

### 1. 到底什么是“人工改写成 ReAct 格式”？

大语言模型（像 GPT-4、PaLM）本质上是一个“超级文本接龙机器”。你给它一段上文，它就会模仿上文的语气和格式，把下文补全。

作者做的事情，就是人工手写一段很长很长的文本。在这段文本里，作者扮演了一个正在解决问题的人类，把**“脑子里想的（Thought）”**、**“手上做的（Action）”**和**“眼睛看到的（Observation）”**，一行一行地以纯文本的形式写下来。

比如，作者从训练集里挑了一个问题，然后**人工敲键盘写出了下面这段文本（这被称为一个 Exemplar/示例）**：

> **Question（问题）:** 苹果公司的现任CEO是哪一年出生的？
> 
> **Thought 1（想法1）:** 我需要先搜索苹果公司的现任CEO是谁，然后再查找他的出生年份。 **Action 1（动作1）:** Search[苹果公司现任CEO] **Observation 1（观察1）:** 蒂姆·库克（Tim Cook）是美国商业高管，自2011年起担任苹果公司的首席执行官。
> 
> **Thought 2（想法2）:** 现任CEO是蒂姆·库克。接下来我需要搜索蒂姆·库克，找出他的出生年份。 **Action 2（动作2）:** Search[蒂姆·库克] **Observation 2（观察2）:** 蒂姆·库克出生于1960年11月1日，美国阿拉巴马州...
> 
> **Thought 3（想法3）:** 观察结果显示他出生于1960年。所以我得到了答案。 **Action 3（动作3）:** Finish[1960]

这就是所谓的**“标准的 ReAct 格式”**。

### 2. 怎么用这些改写好的样本？（少样本提示 Few-shot Prompting）

作者人工写了 6 个这样的完整的问答全过程文本。然后，他们把这 6 个文本**全部拼接在一起**，组成一段超级长的前置文本（这就是 Prompt）。

当有一个全新的问题（第7个问题）需要模型回答时，作者就把这个新问题追加在这个超长文本的最末尾，然后喂给大语言模型：

> _(前面是 6 个完整的人工编写的 ReAct 格式示例)_ ... _(此时开始要求模型回答新问题)_ **Question:** 科罗拉多造山带的东部一直延伸到哪个海拔范围？ **Thought 1:** `[大模型从这里开始自己生成，因为它看了前面的6个例子，学会了这个格式，所以它会开始模仿，输出它的第一个想法...]`
